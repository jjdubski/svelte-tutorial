🎨 Area 4: Custom Themes

Letting users personalize the app's look and feel

Current state: Dark mode toggle works system-wide via a .dark class on <html>. Colors use CSS variables. That's it — one light, one dark.

A

Preset color themes

5-10 curated themes beyond light/dark — e.g., Forest (green), Ocean (blue), Sunset (warm), Midnight (deep dark), Lavender, etc. Each overrides the CSS variable palette.

Effort: Low · Define CSS variable sets + picker UI
B

Accent color picker

Let users pick their own accent color (the blue that's used for buttons, links, highlights). A simple color input or predefined swatches. Saves to localStorage/user settings.

Effort: Low · One CSS variable override
C

Full custom color picker

Users can customize background, card, text, border, button colors independently via color inputs. Saved as a custom theme preset.

Effort: Medium · More UI, more state, more testing
D

Font selection

Pick from 3-4 system-friendly font options (e.g., Inter, System UI, Serif, Monospace). Changes the base font family.

Effort: Low · CSS variable swap
E

Layout density

Compact / Comfortable / Spacious modes that adjust padding, margins, and font sizes in the task list.

Effort: Medium · Many components affected

Click to select. Quick wins vs deeper customization — what feels right?
