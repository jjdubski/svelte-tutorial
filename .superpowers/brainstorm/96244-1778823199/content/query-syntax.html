🔍 Advanced Query Syntax Design

Enhancing the existing search bar to support structured commands

Proposed Syntax

Users type key:value pairs alongside regular text. Regular words still fuzzy-match against title/description.

🔍
tag:work priority:high meeting notes
tag:name priority:high|medium|low is:active|done due:today|tomorrow|overdue|YYYY-MM-DD cat:name sort:date|priority|alpha
t

tag:name

Filter by tag. Multiple tag: values use AND logic (same as tag pills).

Example: tag:work tag:urgent
p

priority:high|medium|low

Filter by priority level.

Example: priority:high
s

is:active|done

Filter by completion status. is:active = not done, is:done = completed.

Example: is:active
d

due:today|tomorrow|overdue|YYYY-MM-DD

Filter by due date. Supports relative keywords and absolute dates.

Example: due:overdue or due:2026-05-20
c

cat:name

Filter by category name.

Example: cat:Work
o

sort:date|priority|alpha|category

Change sort order inline.

Example: sort:priority

Visual feedback: token chips

Parsed key:value tokens appear as colored chips inside the search bar. Plain text stays plain. This gives the user clear feedback that their syntax was understood.

UI filter sync

When a user types priority:high, the priority dropdown in the filter bar also updates to "High". If they then change the dropdown to "Low", the search bar text updates to reflect it. They stay in sync.
